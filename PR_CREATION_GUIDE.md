# How to Create the Pull Request

## 🌐 Step 1: Open GitHub Compare Page

Copy and paste this URL in your browser:
```
https://github.com/Techfourma/discord-bot/compare/main...development
```

---

## 📝 Step 2: Fill PR Details

### Title:
```
refactor: code reorganization & comprehensive documentation enhancement
```

---

### Description/Body:

Copy the entire content from the `PR_TEMPLATE_EN.md` file and paste it into the PR body. 

**Quick shortcuts:**
- The file is located at `/PR_TEMPLATE_EN.md` in the repository
- You can view it in VS Code or any text editor
- Copy all content and paste into the GitHub PR description field

---

## 🏷️ Step 3: Add Labels

Click "Labels" and select:
- ✅ `enhancement` - This is an improvement
- ✅ `documentation` - Documentation was updated
- ✅ `refactoring` - Code was reorganized

---

## 👥 Step 4: Assign Reviewers (Optional)

If you have team members, you can assign them as reviewers by clicking "Assignees"

---

## ✅ Step 5: Create Pull Request

1. Review all the information one more time
2. Make sure the title and body are correct
3. Click the green **"Create pull request"** button

---

## ✨ After Creation

Once the PR is created:
- ✅ Reviewers will get notified
- ✅ GitHub will run automated checks
- ✅ You can discuss changes in the PR comments
- ✅ After approval, merge to `main`
- ✅ Deployment will follow automatically

---

---

## 🎯 What to Expect

Once PR is created, GitHub will automatically:
- Show a timeline of commits
- Display all file changes with diffs
- Run any configured CI/CD checks
- Allow team members to review and comment

The PR will stay open until:
1. All reviewers approve
2. All CI checks pass
3. You click "Merge pull request"

---

## 💡 Tips

- Make sure you're on the correct branches (main ← development)
- The GitHub Compare page will show all changes between branches
- You can edit the PR title and description after creation if needed
- Comments on the PR help with code review discussions
